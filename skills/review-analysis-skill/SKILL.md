# Review Analysis Skill

## Overview

Review Analysis Skill은 쇼핑/커머스 상품 리뷰를 분석하여 판매자와 상품 기획자가 활용할 수 있는 인사이트를 제공하는 Skill이다.

이 Skill은 리뷰 데이터를 입력받아 장점, 불만, 개선사항, 감성 분석, 판매자 액션을 추출한다.

---

## Input

상품 리뷰 목록

예시

```txt
배송이 빨라서 좋았어요.
포장이 꼼꼼해서 만족합니다.
상품에서 쉰내가 나요.
생각보다 크기가 작습니다.
```

---

## Output

```json
{
  "summary": "전체 리뷰 요약",
  "pros": [
    "배송이 빠름",
    "포장이 꼼꼼함"
  ],
  "complaints": [
    "냄새 문제",
    "사이즈 불만"
  ],
  "improvements": [
    "냄새 개선 필요",
    "사이즈 정보 보강"
  ],
  "sentiment": {
    "positive": 35,
    "neutral": 15,
    "negative": 50
  },
  "seller_actions": [
    "제품 냄새 품질 개선",
    "상세페이지 사이즈 정보 추가"
  ]
}
```

---

## Procedure

### Step 1

리뷰 데이터를 수집한다.

### Step 2

긍정 표현과 부정 표현을 식별한다.

예시

- 배송이 빠르다
- 포장이 좋다
- 가성비가 좋다

↓

장점 후보

예시

- 냄새가 난다
- 불량이다
- 파손되었다
- 작다

↓

불만 후보

### Step 3

반복적으로 등장하는 키워드를 그룹화한다.

예시

```txt
배송이 빨라요
배송이 매우 빠름
하루만에 왔어요
```

↓

```txt
배송이 빠름
```

### Step 4

불만 요소를 기반으로 개선사항을 도출한다.

예시

```txt
사이즈가 작아요
```

↓

```txt
상세페이지 사이즈 정보 보강
```

### Step 5

판매자 액션을 생성한다.

예시

```txt
냄새 관련 불만 다수
```

↓

```txt
제품 냄새 품질 점검 및 개선
```

---

## Analysis Categories

### Pros

고객이 만족한 요소

### Complaints

고객이 불만을 가진 요소

### Improvements

제품 또는 상세페이지 개선 가능 요소

### Sentiment

전체 리뷰의 긍정·중립·부정 비율

### Seller Actions

판매자가 우선적으로 수행해야 하는 액션

---

## Constraints

- 반드시 JSON 형식으로 반환한다.
- 장점은 최대 5개까지 추출한다.
- 불만은 최대 5개까지 추출한다.
- 개선사항은 최대 5개까지 추출한다.
- 판매자 액션은 중요도 순으로 정렬한다.
- sentiment 비율의 합은 반드시 100이어야 한다.
